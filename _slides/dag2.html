Bro-øvelse: Hardcoded HTML først

Start med 2-3 film-kort direkte i HTML.

Så mærker I problemet, før JavaScript løser det.

<section id="movie-list" class="movie-grid" aria-label="Filmliste">
  <article class="movie-card">
    <h3>Inception</h3>
    <p>År: 2010</p>
  </article>
  <article class="movie-card">
    <h3>The Matrix</h3>

Hvorfor er hardcoded cards tungt?

  • Du kopierer den samme struktur igen og igen
  • Små ændringer skal laves mange steder
  • 20 film betyder meget håndarbejde
  • Computeren burde gøre det gentagne arbejde for os

Hvad er et array?

Et array er en liste af værdier samlet i én variabel.

Mange variabler

const movie1 = "Inception";
const movie2 = "The Matrix";
const movie3 = "Interstellar";

Én liste

const movies = [
  "Inception",
  "The Matrix",
  "Interstellar"
];
  • Første element ligger på plads 0
  • Arrays er gode når vi har mange ting af samme type

Loop med for...of

const movies = ["Inception", "The Matrix", "Dune"];

for (const movie of movies) {
  console.log(movie);
}

Pointen: Den samme kode kører én gang for hver film i arrayet.

Fra data til DOM

const movieList = document.querySelector("#movie-list");

for (const movie of movies) {
  movieList.innerHTML += `<article>${movie}</article>`;
}

Først viser vi bare titler. Senere gør vi strukturen mere avanceret.

Hvad er et objekt?

Et objekt samler relaterede data med navngivne properties.

const movie = {
  title: "Inception",
  year: 2010,
  rating: 8.8
};

Key: title

Value: "Inception"

Access: movie.title

Access: movie.year

Arrays af objects

const movies = [
  {
    title: "Inception",
    year: 2010,
    rating: 8.8
  },
  {
    title: "The Matrix",
    year: 1999,
    rating: 8.7
  }
];

Nu har hver film flere oplysninger, men vi kan stadig loope gennem listen.

Loop gennem objects

for (const movie of movies) {
  console.log(movie.title);
  console.log(movie.year);
  console.log(movie.rating);
}
  • movie er ét objekt ad gangen
  • Dot notation bruges til at hente properties ud

Hvad er en template literal?

En template literal er en string skrevet med backticks: ` `

Med ${} kan du indsætte variabler direkte i teksten.

const title = "Inception";
const year = 2010;

const text = `${title} kom ud i ${year}`;

Template literal vs. string concatenation

Concatenation

const movieText =
  title + " (" + year + ")";

Template literal

const movieText = `${title} (${year})`;
  • Mere læsbart
  • Mindre rod med +
  • Perfekt til HTML-skabeloner

Byg movie cards med HTML-skabelon

const html = `
  <article class="movie-card">
    <h3>${movie.title}</h3>
    <p>År: ${movie.year}</p>
    <p>Rating: ${movie.rating}</p>
  </article>
`;

Pointen: Én HTML-struktur kan genbruges til alle film.

Indsæt HTML i DOM'en

movieList.insertAdjacentHTML("beforeend", html);

beforeend

Tilføjer HTML sidst inde i elementet.

God vane

Gem HTML i en variabel først. Indsæt bagefter.

Funktioner giver struktur

showMovies();

function showMovies() {
  movieList.innerHTML = "";

  for (const movie of movies) {
    showMovie(movie);
  }
}

En funktion samler én tydelig opgave ét sted.

Én film ad gangen

function showMovie(movie) {
  const html = `
    <article class="movie-card">
      <h3>${movie.title}</h3>
      <p>År: ${movie.year}</p>
      <p>Rating: ${movie.rating}</p>
    </article>
  `;

  movieList.insertAdjacentHTML("beforeend", html);
}

Ekstra: simpelt highlight med betingelse

const highlightClass = movie.rating > 8.5
  ? "movie-card--highlight"
  : "";

Vi bruger data til også at styre udseende. Høj rating kan give særlig styling.
